# 7期计划：火山 403 组级熔断与半开探测

## 目标

当任一火山模型返回 HTTP 403 时，不再让其他火山模型继续重复失败。插件将整组火山模型熔断 30 分钟，期间继续使用非火山 fallback；冷却到期后由后台随机挑选一个仍在 token 安全线内的火山模型做最小探测，成功后恢复整组，失败则重新熔断 30 分钟。

## MVP 范围

- 按 `provider_source_id` 识别火山模型组，当前默认源 ID 为 `openai`。
- 从 Agent 最终错误响应中识别 HTTP 403 / `AccountOverdueError`。
- 在 `quota_state.json` 持久化组级熔断、下次探测时间、触发模型、最近错误和探测租约。
- 熔断期间，路由器把所有火山候选标记为 `provider_group_cooldown`，继续扫描非火山候选。
- 到期后由后台任务随机选择一个仍满足单模型 token 安全线的火山模型做最小文本探测。
- 探测成功清除整组熔断；失败重新设置 30 分钟冷却。
- Plugin Page 和状态 API 暴露组级熔断及探测状态。

## 技术形状

- 数据库：不新增数据库；组级状态继续写入插件 `quota_state.json`。
- 路由：组级跳过与探测候选计算放在 `core/router.py`。
- 状态：原子化组熔断和探测租约放在 `core/state.py`。
- 编排：403 识别与后台探测任务由 `main.py` 接入。
- WebUI：沿用现有 AstrBot Plugin Page，不新增服务。

## 验证

- 火山候选返回 403 后，状态中出现 30 分钟组级熔断。
- 组级熔断期间所有 `provider_source_id=openai` 候选被跳过，非火山模型仍可选择。
- 多次 403 会从最近一次错误重新计算 30 分钟。
- 到期后只有一个探测租约；随机候选必须仍在单模型 quota 安全线内。
- 探测成功清除熔断；探测失败重新冷却且不影响非火山路由。
- 重启后组级熔断继续存在；普通 cache reset 不清除费用保护状态。
- 容器内通过单元测试、compileall、插件加载、状态 API 和日志验证。

## 延后项

- 对插件直接调用 Provider、但不经过 AgentRunner 的 403 做全局拦截。
- 接入火山官方账户或资源包状态 API，替代主动模型探测。
